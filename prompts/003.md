IMPORTANT ARCHITECTURE RULE

ALL authentication related logic must exist ONLY inside:

cxserver/Modules/Auth

Auth logic must NOT be distributed across other modules.

This rule ensures:

• scalability
• maintainability
• clean separation of security concerns

No authentication code should exist inside:

cxcore
cxlib
other modules

Only shared base entities may exist in cxcore.

All security logic must live in Auth module.

Passwords must never be stored as plain text.

Use BCrypt hashing.

Store only:

PasswordHash

Password verification must occur during login.

GenerateRefreshToken
ValidateToken

JWT must include claims:

UserId
Username
Role
Permissions

JWT settings must come from configuration.

---

REFRESH TOKEN ROTATION

Implement refresh token rotation.

Process:

1. Old refresh token is revoked
2. New refresh token generated
3. Stored in database
4. Returned to client

---

AUTH CONTROLLER

Create:

AuthController

Endpoints required:

POST /auth/register
POST /auth/login
POST /auth/refresh
POST /auth/logout
GET /auth/me

Login response must include:

AccessToken
RefreshToken
ExpiresAt

---

LOGIN FLOW

Login process must perform:

1. Validate user credentials
2. Verify password hash
3. Generate JWT access token
4. Generate refresh token
5. Store refresh token
6. Return tokens to client
