OBJECTIVE

Refactor the database naming convention by **removing the "common_" prefix** from all common master tables.

Current tables are too long and verbose.

Example:

common_brands
common_colours
common_contact_types

This should be simplified.

---

NEW NAMING RULE

Tables must be named **directly by entity name**.

Examples:

common_cities → cities
common_brands → brands
common_colours → colours
common_contact_types → contact_types
common_product_types → product_types
common_units → units
common_styles → styles
common_destinations → destinations

Use **snake_case** naming.

---

SCOPE OF CHANGE

The refactor must update:

Database tables
EF Core entity configurations
DbContext mappings
Foreign key references
Indexes
Migrations

Do NOT change entity class names unless required.

Only adjust table mappings.

---

ENTITY CONFIGURATION

Update entity configurations using:

IEntityTypeConfiguration

Example:

Before:

builder.ToTable("common_brands")

After:

builder.ToTable("brands")

Apply this change to all common master entities.

DATABASE MIGRATION

Example:

Ensure foreign keys and indexes are preserved.
