OBJECTIVE

Create a reusable **ListCommon skeleton component** used for all list-style pages.

This component must act as a **page structure template** where all data is loaded dynamically.

Do NOT bind this component to any specific module.

It must work for any dataset.

Examples where it will be used:

users
roles
products
orders
cities
units
brands

But the component itself must remain **generic and reusable**.

---

COMPONENT NAME

ListCommon

Location:

cxstore/src/components/admin/ListCommon.tsx

---

COMPONENT PURPOSE

Provide a standardized **list page layout skeleton** with the following sections:

1. Page Header
2. Search & Filter section
3. Active Filters display
4. Data Table
5. Table Footer (summary)
6. Pagination

All sections must be configurable through props.

---

LAYOUT ORDER

The component must render sections in the following order:

1️⃣ Page Header
2️⃣ Search & Filter Section
3️⃣ Active Filters Section
4️⃣ Data Table
5️⃣ Table Footer
6️⃣ Pagination Section

This order must remain consistent across all pages.

---

SECTION 1 — PAGE HEADER

Header contains:

Page Title
Page Description
Add Button (optional)

Left side:

PageTitle
PageDescription

Right side:

Primary Add Button

Example:

PageTitle → User Management
PageDescription → Manage platform users and roles

Button:

* New Item

Button properties must be configurable.

Props required:

pageTitle
pageDescription
addLabel
onAddClick

---

SECTION 2 — SEARCH & FILTER

Below header create a **Search & Filter section**.

Layout:

Left side:

Search input

Right side:

Filter button
Column visibility selector

Search must support dynamic behavior.

Props required:

searchValue
onSearchChange

Filter options must be passed as props.

Column visibility control must allow hiding/showing columns.

---

SECTION 3 — ACTIVE FILTERS

Below search area display **Active Filters**.

This section only appears when filters or search are active.

Display as:

small tags / chips

Example:

Status: Active
Country: India

Each filter must support removal.

Props required:

activeFilters
onRemoveFilter
onClearAllFilters

---

SECTION 4 — DATA TABLE

Below active filters render the **table container**.

The table content must be passed dynamically.

Do NOT hardcode table columns.

Props required:

columns
data
loading

Use existing table components if available.

If not available create generic table support.

Table must support:

column sorting
responsive behavior
scroll when needed

---

SECTION 5 — TABLE FOOTER

Below the table display a **footer section**.

Purpose:

Display summary information.

Examples:

Total records
Selected rows count
Aggregated totals

Props required:

footerContent

Footer must align with table width.

---

SECTION 6 — PAGINATION

Below table footer display **pagination controls**.

Pagination must support:

page number
page size
total records

Props required:

currentPage
pageSize
totalRecords
onPageChange
onPageSizeChange

---

RESPONSIVE DESIGN

The component must be **fully responsive**.

Mobile layout requirements:

Header stacks vertically

Search section becomes:

Search bar full width

Filter and column buttons move below search

Table must support horizontal scroll

Pagination must remain usable on small screens.

Use TailwindCSS responsive utilities.

---

THEME SUPPORT

Component must work with:

Light mode
Dark mode

Do not hardcode colors.

Use existing theme styles.

---

CONFIGURABLE DESIGN

All dynamic data must come from props.

The component must NOT:

Fetch data directly
Contain business logic
Contain module-specific code

It must only render UI structure.

PROPS STRUCTURE

header configuration

pagination configuration

This ensures flexibility across modules.
