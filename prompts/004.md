LOGIN FLOW

When user logs in:

1. Call backend API

POST /auth/login

2. Receive response:

AccessToken
RefreshToken
ExpiresAt

3. Store tokens securely.

Access token → memory or secure storage
Refresh token → secure storage

4. Load user profile using:

GET /auth/me

5. Update auth state.

---

SESSION PERSISTENCE

On application start:

1. Check if refresh token exists.
2. Call:

POST /auth/refresh

3. Receive new access token.
4. Restore session.

If refresh fails → force logout.

---

ROUTE PROTECTION

Create a route guard.

Location:

cxstore/src/components

Create:

ProtectedRoute.tsx

Responsibilities:

• block unauthorized access
• redirect to login page
• allow authenticated users

---

ROUTING RULES

Public routes → WebLayout

Auth routes → AuthLayout

Authenticated routes → AppLayout

Ensure routing respects authentication state.

---

LOGOUT FLOW

Logout must:

1. Call backend

POST /auth/logout

2. Clear tokens

3. Clear auth state

4. Redirect to login page

---

TOKEN REFRESH

Implement automatic refresh.

When access token expires:

1. Call:

POST /auth/refresh

2. Replace access token

3. Continue session

If refresh fails → logout.

---

FRONTEND SECURITY

Ensure:

• JWT never exposed unnecessarily
• no tokens in URLs
• API requests include Authorization header

Example header:

Authorization: Bearer {token}
